PRIMARY GOAL
Ship a working MVP (web app + API + DB) that runs in Replit with seed data, RBAC, tenant isolation, job posting, applicants pipeline, gig signup/vetting, gig shifts, async interviews, and job board distribution tracking + internal sponsored listings.

- Web: Next.js 14 App Router, TypeScript, TailwindCSS, shadcn/ui
- Forms/Validation: React Hook Form + Zod
- Auth: NextAuth (Credentials email+password). Add optional Google OAuth stub.
- DB: PostgreSQL + Prisma ORM
- Storage (MVP): Local filesystem for uploads (video), behind a storage interface to swap to S3 later
- Jobs/Background processing (MVP): simple in-process queue + cron-like endpoint (design to swap to BullMQ later)
- Tests: Vitest (or Jest) + supertest-like helpers for route handler testing

MONOREPO LAYOUT
- /apps/web  (Next.js app)
- /packages/db (Prisma schema, client, migrations, seed)

- Users can belong to multiple tenants via TenantMembership.
- All tenant-scoped tables MUST include tenant_id.
- All server operations MUST enforce:
  1) authenticated user
  2) membership in tenant
  3) RBAC role authorization
- RBAC roles per tenant: OWNER, ADMIN, HIRING_MANAGER, LOCATION_MANAGER, REVIEWER, VIEWER

  - requireAuth()

  Step B: set travel distance, min hourly rate, availability
  Step C: vetting checklist + attestations (BASIC / VERIFIED / PRO)
  Step D: enable “Gig-Ready” toggle (requires BASIC complete)
- Public gig board at /gigs:
  - list OPEN gigs, filter by role, distance, pay, date

Employer side (tenant-protected):
- Post gig shifts:
  - location, role, startAt, endAt, payRate, requirements (json), acceptanceMode (INSTANT_BOOK / APPROVAL_REQUIRED), emergency flag, status
- Worker acceptance:
  - If INSTANT_BOOK and worker vetted >= VERIFIED (configurable), auto-confirm
  - Else create PENDING request for employer approval
- Gig assignment lifecycle:
  - PENDING, CONFIRMED, CHECKED_IN, COMPLETED, NO_SHOW, CANCELLED
- Reliability score MVP (0–100), starting 100:
  - COMPLETED +2
  - CANCEL within 24h -10
  - NO_SHOW -25
